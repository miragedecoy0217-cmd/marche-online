プロジェクト概要

Overview

本プロジェクトは専門学校の授業内グループワーク（3名チーム）として、マルシェオンライン株式会社（架空）のECサイト刷新を行います。要件定義から動作するシステムの完成・発表まで、98コマ（約6ヶ月）で完成させます。

項目内容
授業時間週4コマ（1コマ1.5時間）＝ 週6時間
残り期間98コマ ÷ 4コマ/週 ≒ 24.5週（約6ヶ月）
総作業時間約147時間（3人合計：441人時間）
開発環境AlmaLinux 9 + Apache + Tomcat + PostgreSQL + HTTPS/TLS
開発手法マイルストーン型（授業進捗に合わせた区切り管理）

DB設計概要（PostgreSQL）

Database
テーブル名概要主なカラム
users会員情報user_id, email, password_hash, role, created_at
products商品マスタproduct_id, name, price, stock, category_id, image_path
categories商品カテゴリcategory_id, name, parent_id
cartカートcart_id, user_id, product_id, quantity
orders注文ヘッダorder_id, user_id, total_price, status, ordered_at
order_details注文明細detail_id, order_id, product_id, quantity, unit_price
reviewsレビューreview_id, product_id, user_id, rating, comment, approved
admin_users管理者アカウントadmin_id, username, password_hash, last_login

設計のポイント

  • 正規化：第3正規形を目指す
  • 在庫管理：DBトランザクションで同時購入対策（SELECT FOR UPDATE）
  • パスワード：bcryptハッシュ化して保存（平文保存は絶対NG）
  • 外部キー：orders→users、order_details→orders など参照整合性を設定
  • インデックス：product.name や category_id に設定
